My work

Integral CRM

I used my experience as a Customer Service rep to really delve into producing the software I wished I got to use. While being made in Sinatra, a simple open source web library for Ruby, I did everything I could to ensure I had maximum functionality in the app. I integrated with Google to enable sign on via an identity provider and handled the routing of this manually. This helped me to understand intimately a key part of the user journey: account creation. The app was styled with W3.CSS, a lightweight CSS framework.

IWANTBOOKS

IWANTBOOKS was focused on creating an efficient way of pulling data from an external API. Using Google Books API and encapsulation to produce a secure way for the app to access the API without interference at the code level, I made sure the app served data in a predictable way. This app was built using Rails and used a server-side strategy to ensure high speed when serving HTML. I also wanted to ensure a seamless experience when using the app, so ensured that any links to Google’s reader do not navigate away from the app. This app also uses OAuth2.0 when authenticating with Google and signing in. This app used Bootstrap 4 for styling.

Cobalt Table Manager

This is somewhat of a capstone project from my learning of React, Redux, and completing my understanding of JavaScript. This app also used a RoR (Ruby on Rails) back end to handle data from the client side appropriately. On the client side, Redux was used to handle state, as the app would handle booking data from restaurant owners so they could manage their restaurants staff. Redux-Thunk was also used to ensure a seamless experience for users when data requests were being processed. I also involved Twilio to send SMS to users who book. Bootstrap 4 was used to style this app.
